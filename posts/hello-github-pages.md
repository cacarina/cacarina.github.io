# 我的第一篇 GitHub Pages 博客

今天我开始搭建自己的个人博客。第一版先不用复杂框架，只用 HTML 和 CSS，这样最容易理解，也最容易发布到 GitHub Pages。

现在我想把写博客这件事变得更简单：以后文章都用 Markdown 来写。

Markdown 的好处是：

- 写标题很方便
- 写列表很方便
- 写链接和代码也很方便
- 不需要每次都手写完整 HTML

## 下一步可以做什么

以后我写新文章时，只需要：

1. 在 `posts` 文件夹里新建一个 `.md` 文件
2. 在 `posts.json` 里登记文章标题、日期和简介
3. 提交并推送到 GitHub

这样博客就能继续更新了。
